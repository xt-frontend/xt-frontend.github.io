AI와 프롬프트만으로 게임 개발 실험에 도전한 사례가 주목받고 있다. 개발자는 1996년 DOS 게임 **Paddlers**를 클론하면서 AI를 활용해 기존 .exe 파일에서 레벨 데이터를 추출하는 독특한 접근을 시도했다. OpenAI Codex CLI와 GPT-5.2 모델을 사용해 "vibe coding" 방식으로 작업했는데, 이는 아이디어 대신 자연스러운 프롬프트로 코드를 생성하는 실험적 방법이다.  

핵심은 기존 게임의 .exe 파일을 데이터 컨테이너로 취급한 점이다. AI가 80KB 크기의 실행 파일을 분석해 **10개 스테이지**와 **11×11 그리드** 패턴을 파악한 과정이 흥미롭다. 개발자는 "첫 스테이지는 5×5 중앙 그리드"라는 단서만 주었고, AI는 패턴 인식을 통해 **0x15FEB** 오프셋에서 레벨 데이터를 정확히 찾아냈다. 이는 바이트 값의 연속성을 분석해 타일 ID를 추출하는 방식으로, AI의 데이터 해석 능력을 극적으로 보여준다.  

이 실험은 게임 개발에서 AI의 잠재력을 시사한다. 기존 게임의 데이터를 재사용하거나, 복잡한 구조를 자동으로 해석하는 것이 가능하다는 점에서, **역공학**이나 **리버스 엔지니어링** 분야에 혁신을 일으킬 수 있다. 개발자 관점에서 중요한 것은 AI가 단순한 코드 생성을 넘어 **문제 해결의 논리 구조**를 이해하는 데 있다는 점이다. 이처럼 AI를 도구로 삼아 창의적 실험을 확장하는 방식은 향후 프로토타이핑, 게임 디자인, 심지어 헤르스테틱(heritage) 게임 보존까지 가능성을 열어준다.
