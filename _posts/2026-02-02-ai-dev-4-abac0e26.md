AI 도구인 Claude Code를 활용한 개발에서 가장 중요한 파일은 프롬프트나 모델 선택보다 `CLAUDE.md`입니다. 이 파일은 AI가 프로젝트에 빠르게 적응하도록 돕습니다.  

개발자는 AI가 세션 시작 시 프로젝트 구조, 패키지 관리자, 언어, 테스트 라이브러리 등을 모르면 실수를 반복합니다. 예를 들어, `pnpm`을 사용하는 프로젝트에서 `npm`을 제안하거나, TypeScript 프로젝트에 JavaScript 코드를 생성하는 등 오류를 유발합니다. 이는 AI 보정에 시간을 소모하게 만들죠.  

`CLAUDE.md`는 프로젝트 루트에 위치하며, AI가 세션을 시작할 때 자동으로 로드됩니다. 이 파일은 AI의 "온보딩 문서"로, 다음 정보를 담아야 합니다:  
1. **프로젝트 개요**: 2~3문장으로 목적과 사용자층을 설명합니다.  
2. **기술 스택**: 언어(TS/JS), 패키지 관리자(pnpm), 테스트 라이브러리(Vitest) 등을 명시합니다.  
3. **폴더 구조**: 소스 코드, 테스트, 설정 파일이 위치한 경로를 정리합니다.  
4. **특이사항**: 레거시 API 대응 코드처럼 개발자만 아는 팁을 포함합니다.  

이 파일을 통해 AI는 프로젝트 맥락을 이해하게 되고, 개발자는 코드 작성 시간을 줄이며 보다 집중적으로 핵심 로직에 투자할 수 있습니다. `CLAUDE.md`는 AI 협업의 효율성을 극대화하는 핵심 자산입니다.
