Wake는 개발 과정에서 발생하는 터미널 내역을 Claude Code가 실시간으로 확인할 수 있도록 돕는 도구입니다. 기존에는 빌드 실패나 디버깅 로그를 설명하거나 복사-붙여넣기해야 했지만, Wake를 통해 터미널의 모든 입력/출력을 클라우드 없이 로컬에 저장해 투명하게 공유할 수 있습니다.

**기술적 접근**  
Wake는 두 가지 핵심 기술을 결합합니다.  
1. **Shell Hooks(preexec/precmd)**: 명령어 실행 시점과 종료 코드를 포착해 터미널 활동의 구조를 파악합니다.  
2. **PTY Wrapper**: 의사 터미널(pseudo-terminal)을 통해 stdout/stderr의 모든 바이트를 실시간으로 기록합니다.  
이 두 방식은 Unix 소켓을 통해 통신하며, Rust의 `portable-pty`와 `tokio`를 활용해 크로스플랫폼 호환성과 비동기 처리 성능을 보장합니다.  

**데이터 관리**  
터미널 내역은 ANSI 코드를 포함한 원시 데이터와 텍스트 형태로 SQLite DB(~/.wake/)에 저장됩니다. 기본 5MB 제한으로 과도한 메모리 사용을 방지하며, 로컬 저장으로 프라이버시를 최우선으로 고려했습니다.  

**Claude Code 통합**  
Wake는 Claude의 Model Context Protocol(MCP)을 통해 터미널 내역을 직접 쿼리할 수 있도록 했습니다. 최근 세션 목록, 명령어 추출, 텍스트 검색 기능을 제공해 개발자와 AI 도구 간의 컨텍스트 전환을 최소화합니다.  

이러한 설계는 복잡한 터미널 로그를 구조화하고, AI와의 협업 효율성을 극대화합니다. 개발자 관점에서 Wake는 단순한 로깅 도구를 넘어, 현대적인 개발 환경에서의 컨텍스트 유지 전략으로서의 가치를 제공합니다.
