```markdown
AI가 Pull Request(PR) 리뷰를 자동화하는 과정에서 발생하는 문제를 해결하기 위해 GitHub CLI(gh) 명령어를 활용한 커스텀 Claude Skill을 개발했습니다. 기존 Claude Code Action을 사용할 때 인라인 댓글이 잘못된 줄에 삽입되거나, 기존 댓글을 조회하지 못해 반복적인 피드백이 발생했습니다. 이는 AI의 확률적 특성으로 인해 불확실성이 높았죠.

본 프로젝트에서 해결한 핵심 문제는 크게 세 가지입니다. 첫째, `gh pr comment` 명령어를 통해 인라인 댓글을 정확한 위치에 삽입하는 방법을 학습시켰습니다. 둘째, `gh pr comments`로 PR 내 모든 댓글(일반 및 인라인)을 수집해 AI가 이력을 기반으로 판단하도록 했죠. 셋째, `gh api` 명령어를 이용해 특정 댓글에 대한 대답을 정확한 컨텍스트에 연결하도록 했습니다. 

이를 위해 Claude Skill Generator를 활용해 gh 명령어 사용법을 체계적으로 가이드하는 프롬프트를 생성했습니다. 예를 들어, "이전 댓글을 고려해 수정을 요청하라"는 지시가 주어지면, Skill이 자동으로 `gh pr comments --limit=100`을 실행해 전체 댓글 목록을 불러옵니다. 이후 AI가 분석한 결과를 `gh pr comment`로 구체적인 줄 수와 함께 제시하게 됩니다. 

개발자에게 주는 인사이트는 AI 도구의 한계를 보완하기 위해 CLI와의 통합이 필수적이라는 점입니다. 단순히 AI에게 작업을 맡기는 것이 아니라, 명확한 API 명령어를 통해 프로세스를 구조화해야 신뢰성 있는 자동화가 가능합니다. 이 경험을 바탕으로, AI와 CLI 툴의 협업은 향후 DevOps 자동화의 핵심 트렌드로 자리잡을 것으로 보입니다.
```
