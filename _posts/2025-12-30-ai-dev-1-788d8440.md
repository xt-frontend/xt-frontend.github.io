###  
AI 개발 도메인에서 "Skills killed MCP"라는 주장이 논란이 되고 있다. 이는 GitHub Actions가 Bash를 대체했다고 주장하는 것만큼이나 과장된 해석이다. MCP(Machine Capabilities Platform)는 AI 에이전트가 실제 작업을 수행하는 핵심 실행 계층이다. Shell 명령, 파일 편집, API 호출, DB 쿼리, 메모리 관리 등은 모두 MCP가 처리한다. 반면 Skills는 작업 흐름을 구성하는 **언어적 표현**에 집중한다.  

예를 들어, GitHub Actions는 YAML로 워크플로우를 정리했지만, 결국 Bash 명령을 실행한다. 마찬가지로, Skills는 "이렇게 실행하세요"라는 지시를 구조화할 뿐, 실제 실행은 MCP가 담당한다. MCP 서버는 서비스 형태로 실행되며, Skills가 지정한 작업을 API 호출로 전달해 실제 코드를 실행한다.  

개발자 관점에서 중요한 점은 **기술의 계층 분리**를 이해하는 것이다. Skills는 팀 내 공유와 재사용을 용이하게 하는 인터페이스 역할을 하며, MCP는 백엔드에서 실행 로직을 제공한다. 두 기술은 상호보완적인 관계다. MCP 없이 Skills는 단순한 '말만 하는' 추상화일 뿐, 실제 작업을 수행할 수 없다.  

결국, "Skills killed MCP"는 표현 방식의 혁신을 과소평가하는 오해다. AI 시스템에서 실행과 표현은 별개의 문제이며, 이 둘의 조화가 비로소 의미 있는 작업을 가능하게 한다. 개발자는 이 관계를 명확히 이해해, 적절한 도구를 계층별로 활용해야 한다.
