AI 기술의 진보에 따라 MCP(Machine Capabilities Protocol)가 사라졌다는 주장이 등장하고 있지만, 이는 과장된 해석이다. 최근 "Skills killed MCP"라는 말이 돌고 있지만, 이는 GitHub Actions가 Bash를 대체했다고 말하는 것만큼 부정확하다. Bash는 여전히 실제 작업을 수행하는 핵심 도구라면, GitHub Actions는 작업 흐름을 더 명확하게 기술하는 표현 방식일 뿐이다. 마찬가지로 Skills는 MCP의 실행을 대체하는 것이 아니라, 작업을 정의하고 공유하는 방법을 개선한 것이다.  

MCP는 AI 에이전트가 실질적인 작업을 수행할 수 있도록 하는 핵심 기반 인프라다. 쉘 명령 실행, 파일 편집, API 호출, 데이터베이스 조회 등 실제 세계와 상호작용하는 모든 작업은 MCP를 통해 처리된다. 예를 들어, GitHub API를 조회하거나 Slack 메시지를 보내는 작업은 실제 인증 정보와 실행 환경이 필요하다. 이는 단순한 지시문으로는 불가능하다.  

Skills는 이러한 작업을 YAML과 같은 구조화된 방식으로 표현하는 인터페이스 역할을 한다. 마치 CI/CD 파이프라인에서 스크립트를 YAML로 정리해 가독성과 공유성을 높인 것처럼, Skills는 작업 흐름을 명확히 정의하지만, 실제 실행은 여전히 MCP가 담당한다.  

개발자 관점에서 두 기술은 상보적이다. MCP는 "무엇을 할지"를 결정하는 실행 엔진이고, Skills는 "어떻게 표현할지"를 정의하는 메타 언어다. 단순히 "Skills가 MCP를 대체했다"고 보는 건 표현 방식과 실행 방식을 혼동하는 것이다. 진정한 가치는 두 기술이 협력하여 개발 효율성을 높이는 데 있다.
