서론  
PDF를 구조화된 데이터로 변환하는 시스템은 복잡한 레이아웃과 오랜 OCR 처리 시간으로 인해 도전적이다. 클라우드 네이티브 아키텍처를 활용하면 확장성과 안정성을 동시에 달성할 수 있다. 본문은 Cloudflare Queues와 Workflows를 기반으로 구축한 문서 파서 파이프라인을 소개한다.  

본론  
파이프라인은 HTTP API → Queue → Workflow → Mistral OCR → Database로 구성된다.  
- **Queues**는 트래픽 스파이크를 완충하고 일관된 배치 크기를 유지한다.  
- **Workflows**는 장시간 실행되는 OCR 작업을 오케스트레이션하며, 실패 시 자동 재시도와 상태 유지 기능을 제공한다.  
- **Mistral AI**의 배치 OCR은 병렬 처리를 통해 대량 문서를 효율적으로 처리한다.  
- **Drizzle ORM**과 **Zod**를 활용해 데이터 검증과 DB 저장을 안정적으로 관리한다.  

개발자 관점에서 주목할 점은 모듈 간 느슨한 결합이다. 각 구성 요소가 독립적으로 재시도 가능하므로 전체 작업이 중단되지 않는다. 예를 들어 OCR 실패 시 해당 작업만 재시도하고 나머지 단계는 유지된다.  

결론  
Cloudflare의 서버리스 스택은 글로벌 배포와 예측 가능한 비용 구조를 제공한다. 이 아키텍처는 인보이스 파싱뿐 아니라 다양한 문서 유형 처리에도 적용 가능하다. Workflows의 오케스트레이션과 Queues의 탄력성은 복잡한 배치 작업의 핵심이며, 개발자는 이를 기반으로 안정적인 시스템을 구축할 수 있다.
